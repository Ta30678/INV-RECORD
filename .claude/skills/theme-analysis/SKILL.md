# 題材分析（theme-analysis）

輸入一個題材名稱，產出一份可長期複盤的題材筆記。目標讀者是筆記作者本人，重點是「這題材為什麼成立、時間點到了沒、誰是買方、誰是指標股、風險在哪」，每個結論都要能追回來源與時點。

## 產出物

- 主檔：`vault/20-題材/<題材名>.md`，frontmatter `type: theme`。
- 連帶更新：`vault/20-題材/題材 MOC.md`（把新題材加進「進行中」清單）。
- 連帶更新：被列為指標股、且**已存在個股筆記**者，在其 `up:` 追加本題材 wikilink。

## 開始前必讀（唯一權威，勿憑記憶）

1. `docs/圖表語法.md` — invchart 區塊語法與圖表選型規範（凍結版，圖表一律照它）。
2. `docs/資料格式.md` — 筆記層級 frontmatter、`up:` 多題材寫法、台灣時間慣例。
3. 至少一份既有題材筆記當結構樣板：`vault/20-題材/先進封裝 CoWoS.md`（六面向最完整）、`vault/20-題材/光通訊 CPO.md`、`vault/20-題材/成熟製程.md`。
4. `CLAUDE.md` — 專案三層結構與領域規則。

## 筆記讀寫工具

優先用 `obsidian:obsidian-cli` skill（省 token、走 metadataCache）；Obsidian 未開或指令失敗時，退回一般檔案工具（Read/Write/Edit），路徑用絕對路徑。

常用指令（第一個參數用 `vault=<你的 vault 名>` 指定 vault；名稱含空格要加引號）：
```bash
obsidian search query="<題材名>" limit=10          # 查題材/個股是否已存在
obsidian read file="先進封裝 CoWoS"                 # 讀結構樣板或既有個股
obsidian create name="<題材名>" content="..." silent # 建新題材筆記（silent 不自動開啟）
obsidian property:set name="up" value="..." file="<個股筆記名>"  # 回補個股 up:
obsidian backlinks file="<題材名>"                  # 完成後確認反查連結
```
一般檔案工具走位時，主檔寫到 `vault/20-題材/<題材名>.md`。

## 工作流程

### Step 0　定位
- 先 `search` 確認同名題材是否已存在。已存在就改為**增修**（保留既有內容、只補查證後的新資訊與時點），不要整份覆寫。
- 讀一份結構樣板，照抄章節骨架（見「筆記結構」）。

### Step 1　研究六大題材邏輯面向
逐一查證下列六節，每節都要有「數字＋來源＋資料時點」。用網路搜尋（WebSearch/WebFetch；需要更深時可用 `deep-research` skill）。

1. **基本面現況** — 技術成熟度、量產/放量現況、主要廠商近況、產能/供需數字。
2. **應用場景與必要性** — 用在哪些終端；「為什麼非用不可」講到物理/經濟瓶頸，而非口號。
3. **是否是未來必須發展/使用的技術** — 給明確結論；列出替代技術與它們分食的空間，區分「技術範疇無可替代」vs「個別供應商可被替代」。
4. **時間點** — 現在是等待期、放量初期還是成熟期；用出貨/產能/客戶路線圖三個以上硬指標互相印證，標關鍵年份節點。
5. **市場規模** — 逐年金額或量（片數/出貨量）與成長率（CAGR）；各研究機構口徑常差一個數量級，先講口徑再講數字，分歧大時採保守值並註明。
6. **主要買方** — 分層寫：直接下單者（晶片/平台商）與終端出錢者（雲端/車廠等）；給集中度（誰佔幾成）與台灣供應鏈對應環節。

查證紀律（全篇適用）：
- **每個數字附來源與資料時點**（來源機構＋日期，如「TrendForce 2026-06-15」）。
- **關鍵數字兩個以上來源交叉比對**；來源打架時兩個都列，說明差異，取保守或標區間。
- **查無資料明確標「無資料」**，不得腦補、不得用約略值假裝精確。
- 媒體轉述、第三方調查、單一分析師觀點要標明性質，別當成官方確認。

### Step 2　挑指標股／龍頭股
- 每檔一條：**wikilink** ＋（交易所別）＋一句定位，下一行「理由」講清楚它為何是這題材的指標/龍頭，附來源與時點。
- **wikilink 對齊既有個股筆記檔名**：先在 `vault/30-個股/` 找是否已有該股筆記，有就用它的確切 basename（例：`[[台積電 2330]]`、`[[環球晶 6488.TWO]]`、`[[博通 AVGO]]`、`[[中芯國際（SMIC） 0981.HK]]`）。
- **尚未建立個股筆記者**：連結照留（點開是空白新筆記屬正常現象），**不要**在這裡順手建立空個股筆記；改在區塊末加一句待補說明（見骨架）。
- 命名慣例（新股待補時）：`名稱 代號`；上櫃股代號加 `.TWO` 後綴（與 canonical ticker 一致），美股/港股用當地代號。
- 清單註明研究日期與「非固定名單、需持續驗證」。

### Step 3　風險
- 依重要性排序，第一條標「最大風險」。涵蓋：需求/資本支出週期、客戶集中、技術棒次交接、競爭分流、上游瓶頸、地緣政治/關稅、估值過高。
- 末段做「資料品質保留」：哪些數字來自第三方調查/媒體轉述而非官方、哪些項目查無資料。

### Step 4　標準圖表組（有資料才畫）
圖表一律用 ` ```invchart ` 區塊，**完全遵守 `docs/圖表語法.md`**（該文件為唯一權威，本節僅摘要）。骨架與可套用範例見 `references/題材筆記骨架.md`。

標準四張（缺資料就略過該張，不要硬湊）：

| 圖 | 圖型 | 放在哪一節 |
|---|---|---|
| 市場規模逐年 | `bar`（離散年度比較） | 市場規模 |
| 需求／出貨成長 | `line`（時間趨勢）或 `bar` | 時間點 或 基本面現況 |
| 產能擴張逐年 | `bar` | 基本面現況 或 時間點 |
| 買方或市佔分配 | `pie`（≤7 項單一時點佔比） | 主要買方 |

圖表鐵律：
- **圖前表後**：圖只輔助閱讀，精確數字仍放緊接其後的表格（圖＋表並存）。
- **每張圖必填 `source` 與 `asOf`**；數據只能來自筆記中已查證附來源的數字。
- **區間值取中點入圖**（如 12~13 萬片取 12.5），並在 `source` 或表格註明原始區間。
- 一篇 **3~5 張圖為上限**，只畫會改變理解的資料；資料點少於 3 個、或表格一眼可讀的不畫。
- 中性資料（營收、產能、市佔）用內建色盤，**不套紅漲綠跌**（那是 K 線慣例）。
- `data` 長度必須等於 `labels` 長度；`bar-line` 的每個 series 要標 `type`。

### Step 5　組裝與寫檔
- 依「筆記結構」組好全文，frontmatter 照骨架。
- `created` 填**台灣時間（UTC+8）當下**，格式 `"YYYY-MM-DD HH:mm"`（帶引號）；`updated: ""`；`up: ""`（題材連到總經為選填，有明確總經母題再填）。
- 標題章節「## 指標股／龍頭股（<研究日期> 研究，個股深度分析待補）」的日期用台灣今天。
- **筆記結尾固定加**：`本筆記僅供個人研究參考，非投資建議。`
- 用 obsidian-cli `create`（或退回檔案工具）寫到 `vault/20-題材/<題材名>.md`。

### Step 6　更新題材 MOC
- 讀 `vault/20-題材/題材 MOC.md`，在「## 進行中」清單加一行 `- [[<題材名>]]`（已在清單就跳過，勿重複）。

### Step 7　回補既有個股的 up:
- 只針對「指標股中**已存在個股筆記**」者：在其 frontmatter `up:` **追加**本題材 wikilink。
- `up:` 是單值就改成 YAML 行內陣列，是陣列就在末端加一個元素：
  ```yaml
  up: ["[[AI 半導體]]", "[[先進封裝 CoWoS]]"]
  ```
- **只追加、不移動檔案**：主題材（`up:` 第一個、決定檔案歸屬）是否改為本題材、要不要把檔案搬到本題材資料夾，由使用者決定，不要自行搬移或重排。
- 尚未建立個股筆記者：不動作（連結留在題材筆記待補即可）。

## 筆記結構（章節骨架）

```
---
type: theme
created: "YYYY-MM-DD HH:mm"   # 台灣時間
updated: ""
up: ""                        # 選填，連到總經母題
tags: [題材]
---

# <題材名>

## 題材邏輯
### 基本面現況
### 應用場景與必要性
### 是否是未來必須發展/使用的技術
### 時間點
### 市場規模          ← 市場規模 bar 圖＋表
### 主要買方          ← 買方/市佔 pie 圖＋表

## 指標股／龍頭股（<研究日期> 研究，個股深度分析待補）
（wikilink 清單 ＋ 每檔理由 ＋ 待補說明 blockquote）

## 風險
（依重要性排序，末段資料品質保留）

## 參考來源（節錄）
（一手來源網址）

---
本筆記僅供個人研究參考，非投資建議。
```

完整可套用骨架（含 frontmatter 與四張 invchart 範例）見 `references/題材筆記骨架.md`——複製後把 `<...>` 佔位換成查證後的內容。

## 完成前自檢

- [ ] 六大面向都有內容，數字都附來源＋資料時點。
- [ ] 關鍵數字有兩個以上來源交叉；查無者標「無資料」。
- [ ] 指標股都是 wikilink，已存在者對齊既有 basename；待補股已在筆記註明。
- [ ] 圖表遵守 `docs/圖表語法.md`：圖前表後、每張有 source/asOf、區間取中點註明、≤5 張、非紅漲綠跌。
- [ ] 結尾有「本筆記僅供個人研究參考，非投資建議。」，日期皆台灣時間。
- [ ] 題材 MOC 已加入本題材。
- [ ] 既有個股 up: 已追加本題材（未移動檔案）。
