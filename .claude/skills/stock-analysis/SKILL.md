# 個股分析（stock-analysis）

輸入：一個股號或公司名。產出：一份可長期複盤的個股研究筆記（`type: stock`），存進 `vault/30-個股/<主題材筆記檔名>/`。

## 動手前必讀

1. `docs/圖表語法.md` — invchart 唯一權威規格（圖型選擇、欄位、錯誤處理）。本 skill 只摘要，衝突時以該文件為準。
2. `docs/資料格式.md` — frontmatter schema、ticker 規則、時間欄位慣例。
3. `CLAUDE.md` — 專案結構與三層筆記關係。
4. 範例結構樣板：`vault/30-個股/**/台積電 2330.md`（用 Glob 找）。深度樣板另見 `上詮 3363.TWO.md`。
5. 本資料夾 `references/範本與圖表.md` — 完整 frontmatter＋內文骨架＋四張標準圖的 invchart 範本，撰寫時照抄再填。

## 共通研究紀律（每篇都要守）

- **每個數字附來源與資料時點**：表格用「資料時點」「來源」欄，行文用括號註明（來源：財報狗 2026-07）。
- **關鍵數字兩個以上來源交叉比對**：股價、EPS、本益比、目標價、營收至少兩來源；不一致時兩個都列並註明差異，取較保守者。
- **查無資料明確標「無資料」**，不得腦補、不得用舊記憶填。標「無資料」不算失敗，是誠實。
- **區間值入圖取中點**並在圖的 `source` 註明「區間值取中點」，精確區間保留在圖後的表格。
- **日期一律台灣時間（UTC+8）**。今天日期見環境提供的 currentDate。
- 筆記結尾固定一行：`本筆記僅供個人研究參考，非投資建議。`

## 筆記讀寫工具

讀、搜尋、就地改既有筆記優先用 `obsidian:obsidian-cli` skill（省 token、走 metadataCache）；Obsidian 未開或指令失敗時退回一般檔案工具（Read/Write/Edit）。路徑一律絕對路徑、含中文與空格加引號；vault 名含空格時第一個參數用 `vault="<vault 名>"`。

```bash
obsidian search query="<公司名或代號>" limit=10        # Step 1 查是否已存在
obsidian read file="<公司名 代號>"                      # 讀既有筆記或結構樣板
obsidian property:set name="updated" value="<台灣今天>" file="<公司名 代號>"  # 就地更新既有筆記 frontmatter
obsidian append file="<題材名>" content="- [[公司名 代號]] …"  # Step 7 回補題材指標股清單
obsidian backlinks file="<公司名 代號>"                 # 交易復盤反查
```

**新建個股筆記例外**：目標路徑是子資料夾 `30-個股/<主題材>/`，而 `obsidian create` 的 `name=` 以 wikilink 解析、不保證落在指定子資料夾——故**新檔一律用檔案工具**（先 `mkdir -p` 建資料夾再 `Write`，見 Step 4~5）。obsidian-cli 專責搜尋、讀取、就地修改既有筆記與反查。

## 工作流程

### 1. 先查是否已存在（存在則更新、不重建）
先用 obsidian-cli `search`／`read`（見「筆記讀寫工具」）查有無本股筆記；不可用時退回 `Glob vault/30-個股/**/*<代號>*.md`（**遞歸**；既有筆記可能扁平放在 `30-個股/`，也可能已重組到 `30-個股/<題材>/` 子資料夾）。
- 找到既有筆記 → **就地更新**該檔（補數字、更新股價快照、`updated:` 填台灣今天、重大改寫投資邏輯才動 `updated`），不要另建、不要搬移路徑（避免斷鏈）。
- 找不到 → 走 2~7 建立新筆記。

### 2. 決定 canonical ticker 與主題材
- ticker 規則（依 `docs/資料格式.md`）：上市純數字（`2330`）；上櫃保留 `.TWO`（`3363.TWO`）；美股代號原樣（`AVGO`）；港股原樣（`0981.HK`）。frontmatter 的 `ticker` 務必加引號。
- 主題材：研究後判定此股最相關的題材，`up:` 第一個為主題材，其餘為次題材。題材筆記在 `vault/20-題材/`，用既有題材檔名（如 `先進封裝 CoWoS`、`光通訊 CPO`）；若該股橫跨多題材用 YAML 行內陣列 `up: ["[[主題材]]", "[[次題材]]"]`。

### 3. 研究與交叉比對
建議來源：財報狗（statementdog）、公開資訊觀測站 MOPS、TWSE/TPEx、Yahoo 股市、TrendForce、公司法說會簡報／逐字稿；美股補 SEC、公司 IR。用 WebSearch/WebFetch 抓取，關鍵數字按上面的紀律交叉比對。查不到就標「無資料」。

### 4. 決定檔案位置
- 新筆記路徑：`vault/30-個股/<主題材筆記檔名>/<公司名 canonical-ticker>.md`
  - 例：主題材 `先進封裝 CoWoS` → `vault/30-個股/先進封裝 CoWoS/台積電 2330.md`。
  - 檔名慣例＝`公司名 ticker.md`（如 `台積電 2330.md`、`上詮 3363.TWO.md`、`博通 AVGO.md`）。
- 子資料夾不存在時**先建立**再寫檔（Bash `mkdir -p`，路徑含中文與空格一律用絕對路徑與引號）。

### 5. 撰寫筆記
新筆記用檔案工具寫入 Step 4 的路徑（`obsidian create` 無法穩定落點到子資料夾，見「筆記讀寫工具」）；更新既有筆記優先用 obsidian-cli `property:set` 或 `Edit` 就地改。照 `references/範本與圖表.md` 的骨架寫。frontmatter 最小集：
```yaml
---
type: stock
ticker: "2330"        # 加引號；上櫃保留 .TWO；美股原樣
name: 台積電
created: "YYYY-MM-DD HH:mm"   # 台灣時間；建立時填
updated: "YYYY-MM-DD HH:mm"   # 同上；日後重大改寫才更新
up: ["[[主題材]]", "[[次題材]]"]   # 第一個為主題材
tags: [個股]
---
```
內文區塊順序（見範本）：K 線 → 股價快照（交叉比對表）→ 投資邏輯 → 基本面檢視 → 產業護城河 → 估值重點 → 風險 → 關鍵交易復盤 → 免責。

### 6. 圖表（有資料才畫，圖前表後）
- 筆記**開頭**放兩個 kline 區塊：`D`／`1y` 與 `W`／`2y`（見範本，ticker 用 canonical 形態）。
- 標準圖表組（invchart，**每張圖都必須先有可交叉查證的數字**；查無資料就跳過該圖，不硬湊）：
  1. **營收結構** → `pie`（單一時點營收佔比，≤7 項）
  2. **月營收＋YoY** → `bar-line`（月營收左軸 bar、YoY 右軸 line）
  3. **毛利率逐季** → `line`（趨勢）
  4. **法人目標價** → `bar` + `horizontal: true`（券商多、名稱長）
- 規矩：每張圖**放在對應資料表格前面**（圖輔助、表精確）；一篇 3~5 張為上限；中性資料用內建色盤，**不**套紅漲綠跌；`source`／`asOf` 必填；區間值取中點並註明。圖型選擇與欄位以 `docs/圖表語法.md` 為準，範本只是常見組合。

### 7. 回補題材筆記的指標股連結
完成後打開主題材（與各次題材）筆記的「指標股／龍頭股」清單（obsidian-cli `read`／`append`，見「筆記讀寫工具」）：
- 若清單已有 `[[本股]]` → 不動（或把「個股深度分析待補」註記更新為已完成）。
- 若清單漏了本股而本股確實是該題材指標 → 補一條 `[[公司名 ticker]]`（附一行理由與來源，比照既有條目格式）。
- 只補連結與精簡理由，**不要**在題材筆記裡重寫整段個股分析。

## 完成檢查清單
- [ ] 既有筆記已就地更新，或新筆記建在 `30-個股/<主題材>/`（資料夾先建好）。
- [ ] frontmatter：`type: stock`、`ticker` 加引號且形態正確、`up:` 首項為主題材、`created`/`updated` 為台灣時間。
- [ ] 開頭有 D/1y 與 W/2y 兩個 kline 區塊。
- [ ] 股價快照表股價至少兩來源交叉比對。
- [ ] 每個數字有來源＋時點；查無者標「無資料」。
- [ ] 圖表：有資料才畫、圖在表前、invchart 欄位齊全、非紅漲綠跌、區間取中點已註明。
- [ ] 結尾有「本筆記僅供個人研究參考，非投資建議。」
- [ ] 已回頭檢查題材筆記指標股清單，需要補的連結已補。
