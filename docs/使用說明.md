# 使用說明

## 三層筆記法

vault 以三層架構組織投資思考，靠 frontmatter 的連結欄位串起來：

```
總經（10-總經/）        國際局勢、央行利率決策
  ↑ up:
題材（20-題材/）        產業題材、領域
  ↑ up:
個股（30-個股/）        投資邏輯、基本面、K 線
  ↑ stock: / theme:
交易（40-交易紀錄/）    每一筆買賣與原因
```

- 每篇筆記用 `up:` 指向上一層（`"[[上層筆記名]]"`），Graph View 就會自動呈現階層
- 交易筆記用 `stock:` 與 `theme:` 同時連到個股與題材
- 個股筆記的 `up:` 可橫跨多個題材（例如台積電同屬 AI 半導體與先進封裝），交易筆記的
  `theme:` 則維持單一值——一筆交易只歸因到促成這次進出的主論點
- Graph View 顏色：🔵 總經、🟣 題材、🟠 個股、🔴 交易紀錄

### 如何看反向連結（完整清單不用手動維護）

個股筆記不會列出「相關交易」全清單，題材筆記也不會列出「相關個股」全清單——
這種窮舉清單只要漏更新一次就會失同步，比沒有清單更危險。完整清單一律交給
Obsidian 原生功能：

- **Backlinks 面板**（筆記右側或下方）：列出所有連到目前筆記的其他筆記
- **Local Graph**：視覺化呈現目前筆記的鏈結網路

個股筆記保留的是「## 關鍵交易復盤」——只精選那 2、3 筆真正改變你判斷的交易並附教訓，
這是 backlinks 給不了的策展價值，兩者互補而非重複。

## 時間紀錄

每篇筆記（總經／題材／個股／交易）frontmatter 都有 `created`（建立時間）與
選填的 `updated`（重大修訂時間），格式為 `"YYYY-MM-DD HH:mm"`、一律台灣時間、
24 小時制：

- `created` 回答「這個投資判斷是何時形成的」——用指令建立的筆記會自動填入，
  用模板手動建立時透過 Obsidian 的 `{{date}} {{time}}` 變數帶入
- `updated` **不會自動維護**，由你在重大改寫投資邏輯時手動填寫；日常的最後編輯
  時間交給 Obsidian 檔案的修改日期即可，不需每次都刷新 `updated`（否則會淹沒
  真正的論點修訂日）
- 交易筆記另有選填的 `time`（成交時間 `HH:mm`），與 `created`（筆記建立時間）
  是兩件事：`time` 是真實成交時刻，`created` 是你動筆記錄的時刻，兩者常有落差
  （例如收盤後才補登當天的交易）

全案時間顯示一律 24 小時制絕對時間（`YYYY-MM-DD HH:mm`、同年可省年份的
`MM-DD HH:mm`、或純日期 `YYYY-MM-DD`），不使用「3 分鐘前」這類相對時間，
方便事後對帳與截圖引用。

## K 線圖（```kline 區塊）

在任何筆記中插入：

````markdown
```kline
2330
period: D
range: 1y
```
````

| 參數 | 說明 | 值 |
|------|------|-----|
| 第一行 | 股票代號（必填） | `2330`、`2330.TW`、`0050` |
| `period` | K 線週期，預設 D | `D` 日 / `W` 週 / `M` 月（也接受中文：日、週、月） |
| `range` | 時間範圍，預設用外掛設定值 | `1mo` `3mo` `6mo` `1y` `2y` `5y` `max` |

圖表上方有 **日K / 週K / 月K** 按鈕可即時切換，⟳ 可強制更新報價。
資料來自 Yahoo Finance；上市股票自動補 `.TW`，上櫃請寫完整代號如 `6488.TWO`。
顏色預設台式**紅漲綠跌**，可在外掛設定改成美式。

標題列下方會顯示「**資料截至**」：這是最後一根 K 棒的日期，答覆「圖上有沒有包含
最近一個交易日」——盤中抓取會標示（盤中），收盤後標示（收盤）。若 Yahoo 連不上、
顯示的是上次快取資料，會改成「⚠ 快取資料｜截至 …｜抓取於 …」並附上實際抓取時間，
讓你判斷手上的舊資料是幾分鐘前還是幾天前的。

## 交易紀錄

一筆交易一個 `.md` 檔，放在 `40-交易紀錄/`。建議用指令建立：

> `Ctrl/Cmd + P` → **INV Record: 新增交易紀錄**

輸入日期、代號、買賣、股數、成交價後，手續費與證交稅會自動試算（可手動覆蓋），
並依 `YYYY-MM-DD 代號 買進/賣出.md` 命名建檔。

「成交時間」欄位選填（`HH:mm`，24 小時制）。同一天有多筆交易時，FIFO 損益會優先
依成交時間排序（缺 time 者一律視為當日最晚）；只有在同日、同一時刻仍分不出先後時
才會退回建檔序號，且同鍵時買進一律先於賣出處理。若你是收盤後一次補登好幾筆交易，
建議填上真實成交時間，才不會讓「先買後賣」被誤判成賣超。

題材欄位選填；`stock:`/`theme:` 連結都是自動反查 vault 中既有的個股／題材筆記
（依 `ticker`／檔名比對），找到才連過去，避免猜錯檔名產生空連結。若還沒有對應筆記，
該欄位會先留空並跳出提醒，建議你用「新增個股筆記」／「新增題材筆記」補上。

frontmatter 欄位見 [[資料格式]]。內文自由發揮——買賣原因、當下情緒、事後檢討。

## 績效儀表板

> 左側緞帶 📈 圖示，或 `Ctrl/Cmd + P` → **INV Record: 開啟績效儀表板**

- **報價更新於**：標題列下方顯示本輪抓取完成的時間（台灣時間），未實現損益這種
  即時數字一定要有時間戳才可信；按 ⟳ 更新報價後這裡會跟著刷新
- **時間範圍**：本月／上月／今年（預設選中）／近一年／全部，五檔按鈕，會記住你
  上次的選擇。「今年」一律指曆年（1/1 起），「近一年」是 rolling 365 天，都不是
  會計年度或滾動 12 個月的近似值
- **摘要卡**（四張）：
  1. **總市值**：目前持倉市值合計
  2. **未實現損益**：目前持倉的未實現損益與報酬率（未實現 ÷ 持倉成本）——這是
     **快照**，永遠用最新報價計算，不受時間範圍影響
  3. **已實現損益（所選範圍）**：範圍內的已實現損益與報酬率（已實現 ÷ 期內平倉
     成本）——只有這張卡受時間範圍約束
  4. **總績效（成立以來）**：只有選「全部」時才顯示數字，公式是
     (累計已實現損益＋目前未實現損益) ÷ 累計買進投入成本；其他範圍顯示
     「—（僅適用於全部期間）」，因為沒有歷史估值就湊不出「期間合併報酬率」，
     假裝精確反而有害（見下方「績效口徑說明」）
- **持倉表**：標題固定寫「目前持倉（以最新報價計，不受上方期間影響）」，
  FIFO 計算的股數、均價（含買進手續費攤入）、Yahoo 現價、未實現損益
- **已實現損益（月）**：只列時間範圍內的月份，每月平倉損益與報酬率
- **個股別已實現明細**：受時間範圍約束，依已實現損益由大到小排序，欄位為
  股票（代號在前，例如「2330 台積電」）、賣出筆數、已實現損益、報酬率、勝率
  （賺錢的賣出筆數 ÷ 總賣出筆數）、平均持有天數（依消耗股數加權，不做年化）。
  股票欄若能在 vault 中找到對應的個股筆記會做成連結，一鍵跳到研究筆記；找不到
  則顯示純文字，不會亂猜檔名
- **資料警告**：frontmatter 格式錯誤、賣超、報價取得失敗、`stock:` 連結與交易
  `ticker` 不一致等，都會列在這裡而不是默默吞掉；若部分報價來自快取，警告會附上
  「最舊為 MM-DD HH:mm 抓取」，讓你判斷快取的嚴重程度

交易筆記存檔後儀表板會自動重算（約 0.5 秒延遲）。

### 績效口徑說明（為什麼不能只看一個百分比）

本外掛沒有現金帳／出入金台帳，也不做歷史 as-of 估值，所以刻意把績效拆成三層，
互不混算：

| 口徑 | 分子 | 分母 | 受時間範圍影響？ |
|------|------|------|------|
| 期間已實現 | 範圍內已實現損益 | 範圍內平倉成本 | 是 |
| 目前未實現 | 未實現損益（快照） | 目前持倉成本 | 否，永遠是「現在」 |
| 總績效（僅全部） | 累計已實現＋目前未實現 | 累計買進投入成本 | 只在「全部」時定義 |

常見誤解：「這段期間明明大賺，怎麼已實現損益卡是 0%？」——如果你這段期間持有沒賣，
損益還沒「實現」，自然不會反映在已實現卡上，去看**未實現損益卡**才是對的。反過來，
若你已經全部出清，未實現卡會歸零，此時已實現卡才是完整的答案。「總績效」只在
「全部」時才有意義，因為此時分子（全部已實現＋現在未實現）與分母（全部買進投入）
的口徑天然一致；其他範圍硬湊一個合併百分比只會誤導判斷，所以刻意留白。

「個股別已實現明細」表格的股票欄若顯示為連結，點擊會直接跳到對應的個股筆記
（依 `ticker` 反查，不是硬拼檔名）；沒有對應筆記時顯示純文字，建議用「新增個股
筆記」補上，之後這裡就會自動變成連結。

## 指令一覽

| 指令 | 功能 |
|------|------|
| INV Record: 開啟績效儀表板 | 在右側欄開啟儀表板 |
| INV Record: 新增交易紀錄 | Modal 輸入 + 費稅試算 + 建檔 |
| INV Record: 新增個股筆記 | 建立含 kline 區塊的個股筆記，題材欄選填（可多個） |
| INV Record: 新增題材筆記 | 建立題材筆記，可選填連到相關總經筆記 |
| INV Record: 新增總經筆記 | 建立總經筆記（自動連 [[總經 MOC]]） |
| INV Record: 清除快取並更新報價 | 強制重抓 Yahoo 報價 |

## 外掛設定

設定 → 社群外掛 → INV Record 投資紀錄：

- **資料夾路徑**：交易／個股／題材／總經筆記的存放位置
- **K 線預設範圍**、**台式/美式配色**、**快取時間**
- **自選股**：沒有持倉也想在儀表板看到報價的股票
- **費用試算**：手續費率、折扣（例如 0.6 = 六折）、證交稅率
