# 使用說明

## 三層筆記法

vault 以三層架構組織投資思考，靠 frontmatter 的連結欄位串起來：

```
總經（10-總經/）        國際局勢、央行利率決策
  ↑ up:
題材（20-題材/）        產業題材、領域
  ↑ up:
個股（30-個股/）        投資邏輯、基本面、K 線
  ↑ stock: / theme:
交易（40-交易紀錄/）    每一筆買賣與原因
```

- 每篇筆記用 `up:` 指向上一層（`"[[上層筆記名]]"`），Graph View 就會自動呈現階層
- 交易筆記用 `stock:` 與 `theme:` 同時連到個股與題材
- 個股筆記的 `up:` 可橫跨多個題材（例如台積電同屬 AI 半導體與先進封裝），交易筆記的
  `theme:` 則維持單一值——一筆交易只歸因到促成這次進出的主論點
- Graph View 顏色：🔵 總經、🟣 題材、🟠 個股、🔴 交易紀錄

### 如何看反向連結（完整清單不用手動維護）

個股筆記不會列出「相關交易」全清單，題材筆記也不會列出「相關個股」全清單——
這種窮舉清單只要漏更新一次就會失同步，比沒有清單更危險。完整清單一律交給
Obsidian 原生功能：

- **Backlinks 面板**（筆記右側或下方）：列出所有連到目前筆記的其他筆記
- **Local Graph**：視覺化呈現目前筆記的鏈結網路

個股筆記保留的是「## 關鍵交易復盤」——只精選那 2、3 筆真正改變你判斷的交易並附教訓，
這是 backlinks 給不了的策展價值，兩者互補而非重複。

## K 線圖（```kline 區塊）

在任何筆記中插入：

````markdown
```kline
2330
period: D
range: 1y
```
````

| 參數 | 說明 | 值 |
|------|------|-----|
| 第一行 | 股票代號（必填） | `2330`、`2330.TW`、`0050` |
| `period` | K 線週期，預設 D | `D` 日 / `W` 週 / `M` 月（也接受中文：日、週、月） |
| `range` | 時間範圍，預設用外掛設定值 | `1mo` `3mo` `6mo` `1y` `2y` `5y` `max` |

圖表上方有 **日K / 週K / 月K** 按鈕可即時切換，⟳ 可強制更新報價。
資料來自 Yahoo Finance；上市股票自動補 `.TW`，上櫃請寫完整代號如 `6488.TWO`。
顏色預設台式**紅漲綠跌**，可在外掛設定改成美式。

## 交易紀錄

一筆交易一個 `.md` 檔，放在 `40-交易紀錄/`。建議用指令建立：

> `Ctrl/Cmd + P` → **INV Record: 新增交易紀錄**

輸入日期、代號、買賣、股數、成交價後，手續費與證交稅會自動試算（可手動覆蓋），
並依 `YYYY-MM-DD 代號 買進/賣出.md` 命名建檔。

題材欄位選填；`stock:`/`theme:` 連結都是自動反查 vault 中既有的個股／題材筆記
（依 `ticker`／檔名比對），找到才連過去，避免猜錯檔名產生空連結。若還沒有對應筆記，
該欄位會先留空並跳出提醒，建議你用「新增個股筆記」／「新增題材筆記」補上。

frontmatter 欄位見 [[資料格式]]。內文自由發揮——買賣原因、當下情緒、事後檢討。

## 績效儀表板

> 左側緞帶 📈 圖示，或 `Ctrl/Cmd + P` → **INV Record: 開啟績效儀表板**

- **摘要卡**：總市值、未實現損益、今年已實現
- **持倉表**：FIFO 計算的股數、均價（含買進手續費攤入）、Yahoo 現價、未實現損益
- **已實現損益（月）**：每月平倉損益與報酬率（= 當月已實現損益 ÷ 當月平倉成本）
- **資料警告**：frontmatter 格式錯誤、賣超、報價取得失敗、`stock:` 連結與交易
  `ticker` 不一致等，都會列在這裡而不是默默吞掉

交易筆記存檔後儀表板會自動重算（約 0.5 秒延遲）。

## 指令一覽

| 指令 | 功能 |
|------|------|
| INV Record: 開啟績效儀表板 | 在右側欄開啟儀表板 |
| INV Record: 新增交易紀錄 | Modal 輸入 + 費稅試算 + 建檔 |
| INV Record: 新增個股筆記 | 建立含 kline 區塊的個股筆記，題材欄選填（可多個） |
| INV Record: 新增題材筆記 | 建立題材筆記，可選填連到相關總經筆記 |
| INV Record: 新增總經筆記 | 建立總經筆記（自動連 [[總經 MOC]]） |
| INV Record: 清除快取並更新報價 | 強制重抓 Yahoo 報價 |

## 外掛設定

設定 → 社群外掛 → INV Record 投資紀錄：

- **資料夾路徑**：交易／個股／題材／總經筆記的存放位置
- **K 線預設範圍**、**台式/美式配色**、**快取時間**
- **自選股**：沒有持倉也想在儀表板看到報價的股票
- **費用試算**：手續費率、折扣（例如 0.6 = 六折）、證交稅率
