# invchart 圖表語法規格（v1.2 凍結版）

本文件是 ` ```invchart ` 程式碼區塊的**唯一權威規格**。外掛實作、分析 skill、筆記撰寫都必須遵守此規格；要改語法先改這份文件。定位比照 `docs/資料格式.md` 之於交易紀錄。

## 設計原則

1. **資料內嵌筆記**：圖表資料直接寫在程式碼區塊內（不外連檔案、不執行查詢），筆記離線可讀、可進版控、複製即帶走。
2. **來源與時點必填**：延續本 vault「每個數字附來源與時點」的紀律，每張圖自動顯示註腳。
3. **圖不取代表**：圖表放在對應資料表格**前面**輔助閱讀，精確數值仍以表格為準（圖＋表並存）。
4. **不發明數字**：圖表數據必須來自筆記中已存在（或已查證附來源）的數字；區間值（如 12~13 萬片）取中點入圖，精確區間保留在表格。

## 區塊語法

區塊內容為 YAML（由 Obsidian `parseYaml` 解析）：

````markdown
```invchart
type: bar-line
title: 台積電 月營收與年增率
labels: [2026-01, 2026-02, 2026-03, 2026-04, 2026-05, 2026-06]
series:
  - name: 月營收
    type: bar
    data: [2932, 2600, 2853, 3495, 3205, 3170]
  - name: YoY
    type: line
    yAxis: right
    data: [35.9, 43.1, 46.5, 48.1, 39.6, 26.9]
unit: 億元
unitRight: "%"
source: 台積電自結月營收（公開資訊觀測站）
asOf: 2026-06
```
````

## 欄位定義

| 欄位 | 必填 | 型別 | 說明 |
|---|---|---|---|
| `type` | ✅ | string | `bar` \| `line` \| `pie` \| `donut` \| `bar-line` |
| `title` | ✅ | string | 圖表標題，顯示在圖上方 |
| `labels` | ✅ | string[] | X 軸類別或圓餅扇區名稱 |
| `series` | ✅ | array | 資料系列（見下）；`pie`/`donut` 恰好一組 |
| `series[].name` | ✅ | string | 系列名稱（圖例） |
| `series[].data` | ✅ | number[] | 數值，**長度必須等於 `labels` 長度** |
| `series[].type` | bar-line 時必填 | string | `bar` 或 `line`（僅 `type: bar-line` 使用） |
| `series[].yAxis` | — | string | `left`（預設）\| `right`（僅 `bar-line` 可用右軸） |
| `unit` | — | string | 左軸／主要單位，顯示於軸與 tooltip（如 `億元`、`%`、`萬片/月`） |
| `unitRight` | — | string | 右軸單位（僅 `bar-line`） |
| `source` | ✅ | string | 資料來源 |
| `asOf` | ✅ | string | 資料時點（如 `2026-06`、`2026 Q1`、`2026-07-09`） |
| `stacked` | — | boolean | 長條堆疊（預設 `false`；用於結構隨時間變化） |
| `horizontal` | — | boolean | 橫向長條（預設 `false`；類別名長或類別多時使用） |
| `colors` | — | string[] | 自訂色票（hex），省略時用內建主題色盤 |
| `height` | — | number | 圖高 px，預設 `300`，允許 `160`~`600` |

## 渲染要求

- **註腳**：圖下方固定顯示 `資料時點：{asOf}｜來源：{source}`，樣式淡化（比照 kline 的資料截至說明）。
- **主題適應**：文字、格線、tooltip 顏色讀取 Obsidian CSS 變數（`--text-normal`、`--text-muted`、`--background-modifier-border` 等），深淺色主題皆可讀；內建色盤需在深淺底上皆有足夠對比。
- **數字格式**：千分位；`unit` 為 `%` 時 tooltip 顯示 `xx.x%`。
- **錯誤處理**：YAML 解析失敗、缺必填欄位、`data` 長度與 `labels` 不符、未知 `type`——一律在區塊位置渲染**繁體中文錯誤訊息**（含具體欄位名），不得 throw、不得整篇筆記渲染失敗。
- **生命週期**：離開預覽時銷毀 chart 實例（不得洩漏）；同一筆記多個區塊互不干擾。

## 圖表選型規範（skill 與筆記撰寫遵守）

| 資料形態 | 圖型 | 範例 |
|---|---|---|
| 佔比／組成（單一時點，≤7 項） | `pie` 或 `donut` | 營收結構、CoWoS 產能分配 |
| 佔比但 >7 項，或類別名很長 | `bar` + `horizontal: true` | 法人目標價比較 |
| 趨勢（時間序列） | `line` | 毛利率逐季、利率走勢 |
| 離散期間比較 | `bar` | 年度市場規模、產能擴張 |
| 量＋率 雙軸 | `bar-line` | 月營收＋YoY%、營收＋毛利率 |
| 結構隨時間變化 | `bar` + `stacked: true` | 各平台營收佔比逐季 |

- 一篇筆記以 **3~5 張圖**為上限，只畫改變理解的資料；少於 3 個資料點、或表格一眼可讀的不畫。
- 中性資料（營收、產能、市佔）用內建色盤；**不**套用紅漲綠跌（那是 K 線的慣例）。

## 範例

**圓餅（營收結構）**

````markdown
```invchart
type: pie
title: 台積電營收結構（2026 Q1）
labels: [HPC, 智慧型手機, IoT, 車用, 其他, 消費性電子]
series:
  - name: 營收佔比
    data: [61, 26, 6, 4, 2, 1]
unit: "%"
source: 台積電 2026 Q1 法說會
asOf: 2026 Q1
```
````

**長條（產能擴張，區間取中點）**

````markdown
```invchart
type: bar
title: 台積電 CoWoS 月產能擴張
labels: [2024 年底, 2026 年中, 2026 年底(目標), 2027 年底(目標)]
series:
  - name: 月產能
    data: [3.5, 7.75, 12.5, 20]
unit: 萬片/月
source: TrendForce 2026-06-15（區間值取中點，精確區間見下表）
asOf: 2026-06
```
````

**橫向長條（法人目標價）**

````markdown
```invchart
type: bar
title: 法人目標價（2026-04~06 發布）
labels: [里昂, 花旗, 匯豐, 野村, 永豐金, 元大/凱基, Aletheia, 高盛]
series:
  - name: 目標價
    data: [3030, 2875, 2820, 2800, 2615, 2600, 2400, 2330]
horizontal: true
unit: 元
source: TechNews 2026-04、豐雲學堂 2026-06
asOf: 2026-06
```
````

## 實作備註（外掛端）

- 解析分兩層：Markdown 處理器用 Obsidian `parseYaml` 得到物件後，交給**純函式** `normalizeChartParams(raw: unknown)`（`plugin/src/charts/blockParams.ts`，不 import obsidian）做驗證與正規化，回傳合法參數或含中文訊息的錯誤——此純函式為 vitest 測試對象。
- 渲染用 Chart.js v4（esbuild 打包進 main.js），註冊 `registerMarkdownCodeBlockProcessor("invchart", …)`，模式比照 `kline/{blockParams,processor,renderer}.ts` 三件式。
